# Jeezlord

A persistent, shared-world, AoE2-style real-time strategy game. One finite map
(512×512 tiles, 32px each) that an always-on server simulates 24/7 — gathering,
building, training and combat advance even while you're logged off. Hardcore
PvP: your base is always vulnerable. Tuned to be slow, so a game arc spans days.

## Status: feature-complete (v0–v7)

A full slow-paced persistent RTS. Implemented and verified end-to-end:

- **Accounts** — register / login over WebSockets (scrypt hashing, `node:crypto`),
  session tokens, auto-reconnect + resume.
- **One shared 512×512 always-on world** — the authoritative 10 Hz server
  simulates 24/7, so gathering, construction, training and combat all advance
  while you're offline.
- **Realtime off-grid movement** with **A\*** pathfinding around obstacles
  (buildings, walls, resource nodes block tiles).
- **Economy** — gather wood/food/gold/stone from scattered nodes; villagers haul
  to drop-site buildings; nodes deplete; renewable **farms** trickle food.
- **Buildings** — house, farm, barracks, archery range, stable, tower, wall, with
  placement ghost, resource costs, and construction that continues offline.
- **Units** — villager, infantry, archer, cavalry, knight, catapult, trained via
  per-building queues with a population cap (houses raise it).
- **Combat** — auto target acquisition, commanded attacks, chasing, attack
  cooldowns, death, tower auto-fire, and wall/building destruction. **Hardcore:**
  bases can be raided while the owner is offline.
- **Fog of war** — you only see within your units'/buildings' line of sight; the
  server never sends out-of-vision enemy entities (anti-cheat enforced at one
  chokepoint). Minimap shows only what you can see.
- **Persistence** — SQLite (`node:sqlite`, WAL) with periodic + shutdown flushes
  and full crash recovery; the world survives restarts.

Pacing is tuned in one file, `shared/stats.ts`. The full design + roadmap is at
`C:\Users\User\.claude\plans\pure-weaving-music.md`.

### Controls
Left-click / drag = select your units · Right-click = move, or gather a resource,
or attack an enemy (context-sensitive) · build/train buttons appear in the bottom
command panel when villagers / a production building are selected · WASD or wheel
= pan/zoom · click the minimap to jump · Esc = cancel placement.

## Tech

- **Server:** Node 22+ + TypeScript (`tsc` → `dist/`), `ws` for WebSockets,
  `node:sqlite` for storage. No external DB, no PM2.
- **Client:** TypeScript + PixiJS v8, bundled with esbuild to `client/bundle.js`.
- **Shared:** `shared/` holds the protocol, constants and types imported by both.

## Develop (Windows)

```
npm install
npm run build        # tsc server + esbuild client
npm start            # serves client + ws on http://localhost:8081
```

Placeholder sprites in `client/assets/` are generated by `npm run gen:assets`
(committed, so you only re-run it after editing `scripts/gen-assets.mjs`).

Or just run `run-dev.bat`. Open `http://localhost:8081` in two browser windows
(one normal, one incognito) to play as two accounts. Watch:
left-click / drag to select your units, right-click to move, WASD / wheel to pan/zoom.

### Tests / smoke checks

Start the server fast-forwarded so multi-second mechanics resolve quickly:

```
TIME_SCALE=40 node dist/server/main.js     # in one shell (fast-forward for tests)

node scripts/smoke.mjs          # register + move (authoritative movement)
node scripts/test-economy.mjs   # gather -> build -> train, pop cap
node scripts/test-combat.mjs    # fog anti-cheat + combat (two players)
node scripts/test-farm.mjs      # renewable farm food
node scripts/test-actions.mjs   # per-unit action + resource amount fields
node scripts/dbdump.mjs 0       # dump persisted rows
```

`TIME_SCALE` multiplies simulation speed (default `1` = real, slow pacing).

## Deploy (root VPS: nginx + systemd)

Build artifacts (`dist/`, `client/bundle.js`) and `data/world.db` are gitignored.

```
# one-time
sudo cp deploy/jeezlord.service /etc/systemd/system/ && sudo systemctl daemon-reload
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/jeezlord  # edit server_name
sudo ln -s /etc/nginx/sites-available/jeezlord /etc/nginx/sites-enabled/
sudo certbot --nginx -d jeezlord.example.com
sudo systemctl enable --now jeezlord

# each deploy
./deploy.sh        # from deploy/deploy.sh.example: git pull, npm ci, build, restart
```

The client connects to `wss://<host>/ws`; nginx serves the static client and
proxies `/ws` to the Node process on `127.0.0.1:8081`. A `systemctl restart`
sends SIGTERM, the server does a final SQLite flush, and clients auto-reconnect.
